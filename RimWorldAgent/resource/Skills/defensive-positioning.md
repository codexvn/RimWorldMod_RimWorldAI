# 防御站位指南

## 核心目标

防御站位不是让所有人排成一排，而是让敌人被迫进入少量接敌格，同时让殖民者的远程火力持续输出。

基本形态：

```text
敌人入口
  ↓
墙 墙 门 墙 墙
      近
沙  沙  沙  沙  沙
远  远  远  远  远

      医
```

- 近战人员站在门口、拐角、窄通道等接敌点，负责缠住敌人。
- 远程人员优先站在沙袋/路障后方，正面对准敌人入口形成稳定射线。
- 墙角探身位只在真实存在可见射线的拐角使用，不能把普通墙后位置当成可射击位。
- 医生和非战斗人员不应该站在射击线上，应留在后方安全区等待救治和支援。
- 所有人都必须有撤退路线，不能被门、墙、敌人或队友堵死。

## 标准流程

1. `enter_plan()` 或暂停游戏，先判断敌人方向。
2. `find_enemies(show_movement=true)` 获取敌人位置、移动方向和预计接敌路线。
3. `get_colonists` 判断谁适合近战、远程、医疗和后勤。
4. `shooting_position_grid` 在敌人接近路线附近扫描远程位。
5. 选择 1-2 个近战堵点，后排选择 2-5 个远程掩体位。
6. `draft_pawn(drafted=true, colonist_ids=[...])` 征召战斗人员。
7. `hold_combat_position(positions=[...])` 批量站位，让近战守点、远程自动开火。
8. 战斗中每 0.5-1 小时或敌人路线变化后重新评估站位。

## 近战站位

近战不是冲出去追人，而是占住敌人必须通过的位置。

### 近战堵门 (Melee Block)

**核心原理**：堵门 ≠ 站门内。站门**外**1 格，让门保持通行能力，3 个 pawn 同时打门口 1 个敌人（1v3）。敌人必须在门口排队进入，后排远程越过近战肩膀输出。

**经典对比（Wiki 实测算例）**：
- 不用堵门：10 疯大象 vs 3 装甲拳师+4 装甲枪手 → 全部 colonist 倒地，10 大象全部存活
- 用堵门：同一场景，3 拳师堵门口 → 只有 1 名 colonist 倒地，全部 10 象消灭
- 结论：堵门是 1 换 10 的效率差，不用堵门 = 全灭

优先位置：
- 单格门口（最佳）
- 墙角外侧、山洞口、桥口、窄通道
- 远程殖民者前方 2-4 格的阻挡位

执行规则：
- 近战穿最高护甲，优先盾带、板甲、动力甲
- 近战站位不能挡住所有远程射线
- 近战只负责缠斗和保护后排，不主动深入敌群
- 使用 `hold_combat_position(role="melee")` 守位
- 只有敌人正在攻击关键目标、落单或需要追杀时，才用 `force_attack(mode="melee")`

**动物堵门**：
- 非栏舍动物（哈士奇/熊/狼）用 `Zone` 限制到门口外 1 格 —— **不需要 Attack 训练**
- 动物自动攻击进入门口范围的敌人，充当肉盾
- 动物会承受大量伤害、很可能死亡 → 只用可牺牲的动物，不用羁绊动物
- 不要用爆炸动物（Boomalope/Boomrat）

**近战轮换**：
- 前排近战受重伤时 → 后退 1 格或进门 → 第二近战顶上堵门位
- 用 `hold_combat_position` 换人，确保堵门位始终有 pawn 占住
- 换下来的 pawn 撤到后方医生处救治

### 拐角伏击 (Wall Ambush)
- 近战躲在拐角/门后断视线
- 敌人接近瞬间冲锋 → 大幅缩短近战暴露距离
- 适用于敌人有远程优势但近战弱的场景
- 任何断视线的角落都可做伏击——门口、室外结构、山丘都有用

### 近战冲锋 (Melee Rush)
- 时机：敌方狙击手/火箭兵在掩体后，超出己方射程时
- 盾带必备——大幅提升冲锋存活率
- 小规模：少量近战冲特定目标（狙击手、火箭兵）
- 大规模：全近战 + 盾带集群冲锋，配合远程火力班同时输出（不怕友伤）
- 冲锋组+远程集火 = 敌人被迫两线应对，效率远高于纯远程
- **不要用近战冲锋对抗敌方近战冲锋——堵门更有效**

### 掩护撤退 (Peeling)
- 当枪手被近战贴脸时：己方近战上前"剥离"敌人仇恨
- 让敌人转火近战 pawn，枪手趁机后退至安全位
- Peel 是高风险操作——剥离子应该是可牺牲或穿重甲的 pawn
- 动物：把'Release animals'设为 Off → 动物自动 peel 主人身边的敌人，不主动追击远处目标

### 远程自卫
- 被近战贴脸的枪手**仍可用枪射击**（只要目标不是正在近战攻击他的 pawn）
- 身体部位武器（动力爪 Power Claw、膝钉 Knee Spike）提升近战自卫伤害
- 高 DPS 枪 + 高品质近战武器组合 = 最大输出

## 远程站位

远程人员必须做到“能打到敌人，但不被敌人轻易贴脸”。

优先位置：

- 沙袋、路障、岩块、树木后方：利用低掩体射击，最稳定
- 门口正后方或斜后方的低掩体位：确保能看到敌人入口
- 墙角相邻格：只在该格确实能看到敌人入口时使用
- 近战堵点后方
- 距敌人预计接敌点 60%-80% 武器射程的位置

执行规则：

- 使用 `shooting_position_grid` 选位，不靠感觉手算坐标。
- 使用 `hold_combat_position(role="ranged")` 到位并进入 `Wait_Combat`。
- 不把 `force_attack(mode="hold_position")` 当成默认防守命令，避免目标离开射线后罚站。
- 远程之间保留间距，避免手榴弹、火箭和爆炸武器一发多伤。
- 短射程武器站前排，长射程武器站后排。
- 墙体会阻挡射线；墙角探身位必须以 `shooting_position_grid` 或实际可射击结果为准，不要按示意图臆测。

## 火力线

布置远程位时要确认三件事：

- 射线没有被墙、门、山体、近战队友完全挡住。
- 敌人进入近战堵点时，至少 2 名远程能同时射击。
- 后排不会因为敌人越过近战位而立刻被贴脸。

典型火力结构：

```text
敌人 → 近战堵点 → 远程交叉火力
```

避免结构：

```text
敌人 → 远程殖民者 → 近战在后面追
```

第二种会导致远程被贴脸、近战追不上、后排罚站。

## 间距与友伤

### 间距管理
- pawn 之间至少 **1 格间距**，减少爆炸/火箭/流弹连锁伤害
- 短射程武器站前排，长射程站后排
- 3 宽排列：pawn 之间隔 1 格，可形成无友伤 3 人横排

### 友伤安全区
- 射手周围 **5 格内完全无友伤**——子弹越过前 2 格（中间隔 1 格）同伴肩膀也不产生友伤
- 5 格外（红区）：子弹未命中后有概率命中友军
- 故事讲述者"友伤倍率"（0-100%）只作用于红区外的概率，**100% ≠ 所有未命中都造成友伤**
- 友伤在近程敌人（狂兽）贴脸时尤其危险 → 敌人贴近后的混乱中友伤可能多于敌人输出

## 火箭应对

- 火箭飞过每 pawn 有**最高 40% 拦截概率**
- **最大化敌方友伤**：让火箭路径穿越尽可能多的敌方 pawn
- 被火箭锁定的近战 pawn：直接**向敌人阵营冲锋** → 要么敌人友伤自爆，要么有机会斩杀火箭兵
- 火箭兵通常站后排准备发射 → 优先狙击或冲锋目标
- 己方用火箭：火箭兵在最优掩体后，其他 pawn 靠前引火但不在火箭路径上

## 救援

- 殖民者倒地后**立即拖出**——不需要直接送医院，先拖到安全区
- 送到安全位置后再回前线，等战斗结束再统一送医
- 高威胁下的重伤员：让救援队吃 **Go-juice** → 加速+抗痛
- **Jump pack** 跳进跳出 → 跳跃期间有 i-frames，跳回最安全
- Locust armor 可替代 jump pack → 牺牲部分护甲但获得机动
- **非战斗人员配盾带+医药**，时刻征召准备现场紧急救治
- 如果时间不够送医（~2 小时内会流血流死）：医生现场先止一点血 → 感染风险高但比死亡好
- 备用方案：未校准的生物塑形舱直接装上伤员

## 撤退与轮换

任何防线都要预留撤退路径。

- 近战受重伤时，后撤一格或进门，第二近战顶上。
- 远程被贴脸时，优先让近战补位，不让远程原地肉搏。
- 医生不站在第一线，战斗结束后立即救治。
- 防线后方不要堆满物品、动物或非战斗人员。

## 机动作战

### 门射击 (Door Potshots)
- 远程 pawn 站门口射击 → 打完一轮 → 后退 1 格关门 → 等冷却 → 开门再射
- **敌人无法还击**——关门瞬间断所有射线
- 多 pawn 可在相邻门重复此操作
- 适合：近程高伤武器（泵动猎枪/链式猎枪），门一开就爆一轮

### 侧翼包抄 (Flanking)
标准 4 步流程：
1. **初始位置**：包抄分队藏在掩体/墙后，与敌人**断视线**
2. **牵制射击**：主力从掩体正面牵制敌人火力
3. **撤退吸引**：包抄分队若被发现 → 撤到墙后，不让接火
4. **绕后攻击**：主力接敌后，包抄分队从侧翼/后方射击

- shift+click 规划绕后路径，避开敌人视线和射程
- 盾带 + 不同方向牵制 + 墙体/地形阻挡都是有效的移动掩护手段

### 射击角度 vs 掩体效率
- **正面射击**：1 个掩体生效（如石块 40%）
- **斜角射击**：2 个掩体同时生效（2 石块总掩护 48%，高于正面）
- **几乎水平方向**：掩体几乎无效（1 石块只有 ~8%）
- 对己方的启示：killbox 中斜角放掩体可获得 **>75% 掩护效率**（墙+沙袋组合在最佳角度可达 ~83-96.75%）
- 敌人一般只有 1 个方向有掩体 → 绕到侧面/后面射击完全废除掩体

### 坦克 (Tanking)
- 盾带 pawn 站在枪手**前排**，吸收敌人火力
- 适合：掩体不足时用 pawn + 盾带补充防线
- 盾带被打掉后**必须立即撤退到掩体后**，等充电恢复
- 注意：高伤单发武器（狙击枪、电荷矛）可一枪**直接破盾**——坦克不要在狙击手射程内暴露

### 打带跑 (Hit and Run)
- 快速 pawn（>120% Moving）+ 长射程武器进入射程 → 射击 → 退出敌人射程 → 重复
- 适合：围攻营地、准备中的袭击者、坠毁飞船——它们不会主动追击
- 一旦敌人进入"全攻"模式或开始冲锋 → 条件不再安全，停止

### 风筝 (Kiting)
- 快速 pawn 在敌人注意范围内持续移动，敌人追不上但也不脱离注意范围
- 其余 pawn **集火**被风筝的敌人
- 武器：突击步枪/栓动步枪（快速长射程）。**不要用狙击枪**——站着瞄准太久
- 轻甲保速度——风筝员不需要重甲，速度是生存关键

**两种变体**：
1. **风筝 #1**（敌人速度 ≈ 己方大多数）：1 个快速 pawn 绕圈吸引全体敌人 → 其余 pawn 在安全距离输出。黑色圆圈 = 理想路线，红色线 = 风筝员太慢时的备选路线
2. **风筝 #2**（所有 colonist 都比敌人快/比敌人射程远）：在敌人进入射程前就后退，始终保持距离

- 如果敌人追上 → pawn 被减速+孤立 → 非常危险。确保风筝 pawn 比敌人快

- 远程人员站得太靠前，被近战敌人直接贴脸。
- 近战人员冲出门口太远，导致后排射线混乱。
- 所有人站成横排，没有堵点和纵深。
- 用 `force_attack` 长时间维持防守，目标移动后殖民者罚站。
- 战斗中使用大步 `advance_tick`，错过敌人突破、倒地和撤退时机。
- 疯狂动物、小型近战敌人来袭时没有全员征召，导致单人被追死。

## 激活时机

- 收到袭击、疯狂动物、虫族、机械族或围攻事件。
- 发现远程人员贴脸、近战没有顶住、殖民者罚站。
- 准备建设防御工事或选择长期防守点。
- 战斗复盘时需要总结站位问题。
- 需要风筝、打带跑、侧翼包抄等机动作战时。
