# 防御站位指南

## 核心目标

防御站位不是让所有人排成一排，而是让敌人被迫进入少量接敌格，同时让殖民者的远程火力持续输出。

基本形态：

```text
敌人入口
  ↓
墙 墙 门 墙 墙
      近
沙  沙  沙  沙  沙
远  远  远  远  远

      医
```

- 近战人员站在门口、拐角、窄通道等接敌点，负责缠住敌人。
- 远程人员优先站在沙袋/路障后方，正面对准敌人入口形成稳定射线。
- 墙角探身位只在真实存在可见射线的拐角使用，不能把普通墙后位置当成可射击位。
- 医生和非战斗人员不应该站在射击线上，应留在后方安全区等待救治和支援。
- 所有人都必须有撤退路线，不能被门、墙、敌人或队友堵死。

## 标准流程

1. `enter_plan()` 或暂停游戏，先判断敌人方向。
2. `find_enemies(show_movement=true)` 获取敌人位置、移动方向和预计接敌路线。
3. `get_colonists` 判断谁适合近战、远程、医疗和后勤。
4. `shooting_position_grid` 在敌人接近路线附近扫描远程位。
5. 选择 1-2 个近战堵点，后排选择 2-5 个远程掩体位。
6. `draft_pawn(drafted=true, colonist_ids=[...])` 征召战斗人员。
7. `hold_combat_position(positions=[...])` 批量站位，让近战守点、远程自动开火。
8. 战斗中每 0.5-1 小时或敌人路线变化后重新评估站位。

## 近战站位

近战不是冲出去追人，而是占住敌人必须通过的位置。

优先位置：

- 单格门口
- 墙角外侧
- 沙袋前一格
- 山洞口、桥口、窄通道
- 远程殖民者前方 2-4 格的阻挡位

执行规则：

- 近战穿最高护甲，优先盾带、板甲、动力甲。
- 近战站位不能挡住所有远程射线。
- 近战只负责缠斗和保护后排，不主动深入敌群。
- 使用 `hold_combat_position(role="melee")` 守位。
- 只有敌人正在攻击关键目标、落单或需要追杀时，才用 `force_attack(mode="melee")`。

## 远程站位

远程人员必须做到“能打到敌人，但不被敌人轻易贴脸”。

优先位置：

- 沙袋、路障、岩块、树木后方：利用低掩体射击，最稳定
- 门口正后方或斜后方的低掩体位：确保能看到敌人入口
- 墙角相邻格：只在该格确实能看到敌人入口时使用
- 近战堵点后方
- 距敌人预计接敌点 60%-80% 武器射程的位置

执行规则：

- 使用 `shooting_position_grid` 选位，不靠感觉手算坐标。
- 使用 `hold_combat_position(role="ranged")` 到位并进入 `Wait_Combat`。
- 不把 `force_attack(mode="hold_position")` 当成默认防守命令，避免目标离开射线后罚站。
- 远程之间保留间距，避免手榴弹、火箭和爆炸武器一发多伤。
- 短射程武器站前排，长射程武器站后排。
- 墙体会阻挡射线；墙角探身位必须以 `shooting_position_grid` 或实际可射击结果为准，不要按示意图臆测。

## 火力线

布置远程位时要确认三件事：

- 射线没有被墙、门、山体、近战队友完全挡住。
- 敌人进入近战堵点时，至少 2 名远程能同时射击。
- 后排不会因为敌人越过近战位而立刻被贴脸。

典型火力结构：

```text
敌人 → 近战堵点 → 远程交叉火力
```

避免结构：

```text
敌人 → 远程殖民者 → 近战在后面追
```

第二种会导致远程被贴脸、近战追不上、后排罚站。

## 撤退与轮换

任何防线都要预留撤退路径。

- 近战受重伤时，后撤一格或进门，第二近战顶上。
- 远程被贴脸时，优先让近战补位，不让远程原地肉搏。
- 医生不站在第一线，战斗结束后立即救治。
- 防线后方不要堆满物品、动物或非战斗人员。

## 常见错误

- 远程人员站得太靠前，被近战敌人直接贴脸。
- 近战人员冲出门口太远，导致后排射线混乱。
- 所有人站成横排，没有堵点和纵深。
- 用 `force_attack` 长时间维持防守，目标移动后殖民者罚站。
- 战斗中使用大步 `advance_tick`，错过敌人突破、倒地和撤退时机。
- 疯狂动物、小型近战敌人来袭时没有全员征召，导致单人被追死。

## 工具选择

| 场景 | 工具 |
|------|------|
| 查看敌人方向 | `find_enemies(show_movement=true)` |
| 扫描射击位 | `shooting_position_grid` |
| 检查防御位 | `defend_position(action="list")` |
| 批量站位待命 | `hold_combat_position` |
| 征召战斗人员 | `draft_pawn` |

| 集火或追杀 | `force_attack` |
| 恢复/减速 | `toggle_pause` |

## 激活时机

- 收到袭击、疯狂动物、虫族、机械族或围攻事件。
- 发现远程人员贴脸、近战没有顶住、殖民者罚站。
- 准备建设防御工事或选择长期防守点。
- 战斗复盘时需要总结站位问题。
