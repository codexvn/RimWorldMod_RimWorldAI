# 战术地图集

以下 ASCII 地图**严格基于 rimworldwiki.com 的实际截图/示意图**转写。
每张图标注来源。

符号说明：
- `＃` = 墙（不可通行/不可射击）
- `门` = 门（1 格宽）
- `近` = 近战 colonist
- `远` = 远程 colonist
- `象` = 大象 / 大型近战敌人
- `敌` = 敌方单位
- `倒` = 倒地 pawn（红叉）
- `血` = 血迹区域
- `→` = 射击方向 / 冲锋方向
- `⬤` = 黑色粗箭头路径（原图手绘）
- `⬥` = 红色箭头（原图手绘）
- `▬` = 黑色路径线
- `···` = 移动路径

## Map-1：堵门正确示范 ✅

来源：`Melee_test_eleplants_block.png`（320×400）
wiki 说明：Same situation but with melee blocking — only 1 colonist downed, all elephants defeated.

```text
             门外面
        象
      血 血 血
    ＃＃门＃＃
    ＃近 近 ＃
    ＃近 近 ＃
    ＃ 远 远 ＃
    ＃＃＃＃＃
       门里面
```

布局：1 格宽门，两侧墙。3 近战站在门外格，形成 3v1 堵口。
敌人（大象）只能通过 1 格门逐个进入，被 3 近战同时打击。
后排远程越过近战肩膀安全输出。

**结果**：3 装甲近战 + 4 装甲远程 vs 10 疯象 → 仅 1 名 colonist 倒地，全歼 10 象。

## Map-2：不堵门错误示范 ❌

来源：`Melee_test_eleplants_normal.png`（400×320）
wiki 说明：10 manhunter elephants vs 3 armored brawlers + 4 gunners — all colonists downed, all elephants remain standing.

```text
    ＃＃＃＃＃＃
    ＃        象
    ＃象 倒倒倒 象
    ＃象 倒血倒  象
    ＃象  倒     象
    ＃         象
       (无门/开放)
```

布局：三面墙围成的半开放空间，**底部没有门**，敌人可从 360° 进入。
7 名 colonist 挤在中央 2×2 格内，10 象从四面八方冲入践踏。

**结果**：全部 colonist 倒地（红叉），全部大象健康站立。

**教训**：没有堵门 = 全灭。堵门 = 1 换 10。

## Map-3：Body Block Choke

来源：`Bodyblock_choke.png`（250×360）
核心：5 名 colonist 堆叠在门格上，物理阻止门打开。

```text
    ＃＃＃＃＃
    ＃      ＃
    ＃ 远远  ＃
    ＃ 远远  ＃
    ＃ 门近  ＃  ← 所有 pawn 堆叠在门格
    ＃      ＃
    ＃      ＃
    ＃＃＃＃＃
      敌人方向 →
```

布局：单格门在墙体中段。门关闭，5 pawn **站在门格上**（或是门外侧+门内侧各堆叠）。
最前排 Diver（持近战+骷髅头盔），后方堆叠 Jan / Thunder / Jono / Steven Trigger。
所有人**面朝门外**。物理占据门格 → 门无法被推开 → 零成本防御。

适用：早期基地守卫、突袭前临时封锁、资源匮乏时替代建材堵口。

## Map-4：拐角伏击

来源：`Melee_ambush.png`（400×216）
wiki 说明：Attacking a ranged opponent using a corner of a hallway.

```text
        纵走廊
        ↓ 敌
        ↓
    ＃＃＃┐＃＃＃
    ＃       ← 横走廊
    ＃
    ＃ 近近  （拐角内侧藏身处）
    ＃ 远远
    ＃
```

布局：L 型走廊 90° 拐角。2 名近战蹲伏在**拐角内侧**（L 的内角），紧贴垂直墙。
敌人从纵向走廊上方下来，视线受墙体阻挡，完全看不见拐角内侧的趴位 pawn。

**执行**：敌人转过拐角进入横向走廊第 1 格 → colonist 只需冲锋 **1-2 格** 即可贴脸（vs 开阔地需要 5-6 格）。
敌人刚转角、视野受限、惯性向前，反应时间极短。

## Map-5：射击角度 × 掩体效率

来源：`Fire_cover_straight.png` / `Fire_cover_angled1.png` / `Fire_cover_angled2.png` / `Fire_cover_angled3.png`
wiki 说明：4 张子图对比不同角度的掩体效率（Alpha 16 数据，机制至今不变）。

```text
正面（40%）     斜角（48%）      更大斜角（37%）   几乎水平（~8%）

  敌               敌              敌             远 → 敌
  ↑                ↗              ↗↗
  石               石  石          石 石         石（几乎无效）
  ↑                  ↗               ↗
  远               远              远
```

- **正面射击**（正对掩体）：1 个掩体生效 → 石块 40%
- **斜角射击**（45°）：2 个掩体同时生效 → 2 石块 48%（反而更高！）
- **更大斜角**：2 掩体生效但角度太偏 → 37%
- **几乎水平**：掩体几乎无效 → 石块仅 ~8%

己方启示：killbox 中斜角放置掩体可 >75% 效率（墙+沙袋组合 83-96.75%）。
敌人一般只有 1 个方向有掩体 → 包抄到侧后方射击完全废除掩体。

## Map-6：风筝 #1 — 单人绕圈 Kiting

来源：`Kiting_1.png`（400×393）
原图含手绘黑色弯曲箭头 + 红色箭头。

```text
    ⬤ 黑色路径（逆时针环绕敌群）

        敌敌敌敌（12-15 个，中央聚集）
      血          血
            ⬤
    远         ⬤
     ⬤              远
   （快速 pawn >120% Moving）
            ⬤

    ⬥ 红色箭头 = 移动方向指令
    ⬥ 底部双人组（深色服装）向上移动
    ⬥ 左中单 pawn 向左上方牵引

    黑色路线 = 理想环绕路径
    红色线 = 太慢时的备选路线（敌人可能追上但仍有输出时间）
```

战术：1 个快速 pawn（>120% Moving）绕敌人集群做**逆时针环绕**，保持自己在敌人的注意范围内。
其余 pawn 在安全距离集火被牵制的敌人。轻甲保速度，武器用突击步枪/栓动。

## Map-7：风筝 #2 — 团队阶梯 Kiting

来源：`Kiting_2.png`（364×400）
含白色锯齿线 + 黑色粗箭头。

```text
              敌
            （灰绿，右上角）
         血   血

    ＃墙＃
    ＃  ← 白色锯齿路线（阶梯状后退）
    ＃  ⬥
    ＃   远 远 远 （3 pawn 编队）
    ＃
        ⬤ 黑色粗箭头 = 下一移动方向（左下）
```

与 Kiting #1 的区别：
- Kiting #1：单 pawn 绕圈
- Kiting #2：**3 pawn 编队** + 阶梯状（之字形）后撤路径
- 白色锯齿线：利用墙体做掩体停顿射击 → 再后撤 → 再停顿
- 黑色粗箭头 ← 下一步移动方向（往左下诱敌）
- 阶梯状前进避免直线暴露，比直线更难被命中
- 比 #1 风险低、容错高，适合对抗更强（慢速）敌人

## Map-8：友伤安全区 + 火箭拦截

来源：`Friendly_Fire_Radii.png`（wiki 页面嵌入图）
wiki 说明：Central pawn will not hit any friendly pawn on or within the gold radius.

```text
        红区（可能友伤，×讲故事者倍率）
      ╱               ╲
    ╱                   ╲
   ╱    金区（无友伤）    ╲
  ╱      ╱         ╲      ╲
 │      ╱   射手    ╲      │
 │     │    ⬤      │     │
 │      ╲   5 格   ╱      │
  ╲      ╲       ╱      ╱
   ╲                ╱
    ╲              ╱
      ╲          ╱
        （无菌砖区域：组队无友伤）
```

- **金区**（5 格内）：完全无友伤。组队站无菌砖区域可互不误伤。
- 可越过前 **2 格**同伴肩膀射击，无友伤产生。
- **红区**（5 格外）：子弹未命中后有概率命中友军，概率受讲述者倍率控制。
- 火箭飞过每 pawn 有**最高 40% 拦截概率**——让火箭路径穿越最多敌人 = 最大化敌方友伤。

## Map-9：围攻营地

来源：`Siege_base_construct.png` / `Siege_base_finished.png`（各 ~400×400）
wiki 说明：Siege camp under construction / Finished siege camp.

```text
    围攻营地（建设中 → 完工）

    建设中：              完工后：
    
     沙 沙 沙              沙 沙 沙
     迫     迫              迫     迫
     （建造中）    →      （已完工，开火）
     弹药箱                 弹药箱
     敌 敌 敌              敌 敌 敌
       （无沙袋掩护处仍暴露）
```

营地布局：2 门迫击炮 + 沙袋围成的阵地 + 补给弹药箱。
- **最佳进攻时机**：营地刚开工，补给已空投但沙袋未完工 → 敌人只能用天然石块做掩体。
- 迫击炮对轰：HE 高爆清人（忽略掩体），燃烧弹牵制灭火，EMP 眩晕迫击炮。
- 灭火弹干扰：空投舱装泡沫灭火弹 → 落到敌营地附近 → 敌人误把泡沫弹装入迫击炮 → 废掉炮位。
- 山体隧道：迫击炮打不到头顶山体下方——挖隧道/避难室。

## Map-10：虫害火烧房

来源：`Infestation_within_mountain_rooms_bugs.png`（400×250）
wiki 说明：封闭房间 → 燃烧瓶 → 温度飙升至 2000°C → 虫巢+虫子全熟。

```text
    ＃＃＃＃石门＃＃＃＃     ← 石门耐火
    ＃ 木家具   虫虫 ＃
    ＃ 燃料      虫巢 ＃
    ＃ 木地板  虫虫虫 ＃
    ＃ 血 IED燃烧     ＃
    ＃＃＃＃＃＃＃＃＃＃＃
    
    关门 → 触发 IED → 温度 ↑ 2000°C
```

- **150-200°C 慢烤**：只烧地面（不直接烧物品/虫子）→ 虫子中暑昏迷，**不会激怒**（无烧伤伤害）
- 虫子严重中暑（>60%）后再派 pawn 进入清场
- 燃料：沾血衣物、干尸、木地板烧得久；大衣/风雪大衣血量高 → 烧更久
- 诱饵房：远离主基地 + 便宜木家具 + IED 燃烧陷阱 → 虫灾发生后自动触发

## Map-11：空投舱防御

来源：`Crashed_ship_part_using_foam_poppers.png`（500×312）
wiki 说明：Mechanoids triggered when part damaged or built within 3-tile radius.

```text
    ＃＃＃＃＃＃＃＃＃＃＃
    ＃                   ＃
    ＃    空降落点        ＃
    ＃   （520 ticks）    ＃
    ＃    石架 石架       ＃  ← 室内预设耐久掩体
    ＃    泡沫灭火器      ＃  ← 防止房间烧毁
    ＃         门         ＃
    ＃ 远→远→远          ＃
    ＃＃＃＃＃＃＃＃＃＃＃

    头顶山体 → 空投无法穿透
```

- 延迟 520 ticks（8.67 秒）→ 不到 9 秒反应时间
- 两种策略：(A) 立即围攻 → 所有就近 pawn 征召找掩体 (B) 等敌人拆家具后逃跑 → 门口堵截
- 头顶山体永远无法被空投穿透 → 山体挖避难室
- 石架做耐久非易燃掩体，泡沫灭火器预先铺设防火
- 室内火烤（关门让火烧）：只对人类有效，机械族免疫温度+无法点燃
